# Udacity Blockchain Capstone

## Introduction

The output of this project is a contract deployed on Rinkeby, that manages the minting and trading of non-fungible tokens. 
Open Sea is used to facilitate trading. 

## ERC-721 Tokens

A UML class diagram is provided in ./eth

The SolnSquareVerifier contract is Ownable and abides to the following interfaces : ERC721, ERC721Enumerable and ERC721Metadata.

SolnSquareVerifier also interacts with a Verifier contract, that must be deployed on the same network.

The Verifier contract is built using Zokrates and based on a simple proof formula. 


## Zokrates

Zokrates was used to define a simple proof formula and generate a set of 10 unique proofs.

These proofs are stored in ./eth/migrations/3_proofs.json for use by unit tests and deployment.

During deployment, these proofs are used to mint 10 tokens.

## OpenSea

I have registered two accounts with Open Sea:
- a seller account, which is the account used for deployment and minting.
- a buyer account, which is used to buy a bundle of 5 properties from seller.

The address to view the listing and sale of that bundle:
https://testnets.opensea.io/bundles/bundle-gTQ/

## Testing locally

3 test files are included in ./eth/test: 
- TestSquareVerifier.js is used to test the Verifier contract
- TestERC721Mintable is used to test the CustomERC721Token contract, which is the parent class of SolnSquareVerifier
- TestSolnSquareVerifier is used to test the SolnSquareVerifier contract - the code uses Zokrates proofs to mint
